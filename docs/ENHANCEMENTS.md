# Enhancements

> **Purpose:** Evidence-backed enhancement audits, invention, and feasibility review.
> **Read when:** Running an Enhancement checkpoint.
> **Update when:** Scope, source policy, or triage changes.
> **Synchronize with:** CHECKPOINTS.md, RESEARCH.md, ROADMAP.md, STATUS.md, ISSUE_TRACKING.md.
> **Status:** Active.
> **Activation:** Standard profile. Run on demand.

## When to run

On demand — when looking for evidence-backed improvements worth scheduling.
There is no step-count schedule (cadence retired 2026-07-18; see ROADMAP.md).
Report: `docs/enhancements/ENHANCEMENT_<checkpoint-id>.md`.

## Method

Treat each run as product evolution: research official docs, maintained repos,
release notes, and comparable portfolios (without copying protected content).
Review fit, value, friction, performance, UI/UX, accessibility, security/privacy,
maintainability, and cost. For each candidate record benefit, evidence, impact,
risk, tests, rollback, and disposition.

Candidate future enhancements (parked, not committed): SEO/social metadata depth,
per-product FAQ, press kit, changelog/devlog, localization, analytics disclosure
(would trigger a privacy-reviewed packet). Reject over-scoped, license-risky,
insecure, or complexity-heavy ideas. Split accepted ideas into single packets.
