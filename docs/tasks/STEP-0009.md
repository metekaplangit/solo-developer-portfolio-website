# Task Card — STEP-0009

- **ID:** STEP-0009
- **Type:** feature
- **One outcome:** Make the privacy and support pages **App-Store-review-ready** by presenting Apple Guideline 5.1.1(i) + 1.5 requirements as clear, explicit, labeled sections, and enforcing the required fields in the content model so an incomplete policy fails the build.
- **Why now:** User is preparing App Store submissions; a clear/complete privacy page + working support contact are among the most common review checkpoints. Reducing rejection risk now avoids rework later.
- **Acceptance criteria:** (1) Privacy schema requires the Apple-mandated pieces — what's collected, **how it's used**, third-party equal-protection confirmation (when applicable), and a **retention/deletion + revoke-consent/request-deletion** statement — such that a policy missing them **fails `npm run build`**. (2) `PolicyArticle` renders these as clear labeled sections (per app), publicly, with last-updated + scope, and an in-app account-deletion note when the app has accounts. (3) Support page keeps a genuine, easy contact (email) with a response expectation (Guideline 1.5). (4) A docs note tells the developer to keep each real app's policy truthful, enable the support mailbox, and add in-app account deletion if accounts exist — with a clear "not legal advice / human-review" caveat. (5) `npm run build`, `npm run check`, `npm test`, and `python3 scripts/validate-governance.py` pass.
- **Non-goals:** Legal guarantee of approval; writing real per-product policies (samples only); analytics/third-party integrations; account systems.
- **Evidence method:** command/log artifact (build/check/test/validator; a negative test that a policy missing `retention` fails validation) + runtime screenshot of a privacy page showing the Apple-checklist sections.
- **Proof classification:** machine-verifiable + manual-runtime (visual).
- **Required checks:** `npm run build`; `npm run check`; `npm test`; `python3 scripts/validate-governance.py`; runtime visual.
- **Issue links:** LEDGER-001, LEDGER-002 open (low, unrelated).
