# Task Card — STEP-0041

- **ID:** STEP-0041
- **Type:** feature (UI — layout rhythm & composition)
- **One outcome:** Vertical spacing carries meaning instead of being one repeated value, the largest dead region on the site is composed rather than empty, and a latent horizontal-overflow bug is closed before a second product can trigger it.
- **Why now:** `/impeccable layout` (dual isolated assessments). The layout-scoped detector returned **0 findings** on both `dist/` and `src/` — which is exactly the failure mode the reference warns about: *"a monotone grid with uniform spacing passes every detector rule, which is exactly what the assessment exists to catch."* The assessment found the site had **two** macro spacing decisions total, and every band gap on every page measured identical (48px ≤872px, 70px at 1280, 72px ≥1309).
- **Acceptance criteria:** (1) A three-step band rhythm (`--band-y-tight` / `--band-y` / `--band-y-loose`) applied per *relationship*, so measured gap sequences vary within a page. (2) The latent `.head-grid` overflow is closed — a single-token product name must not overflow the document at 320px. (3) The footer is pinned. (4) Prose band's dead width is composed or removed. (5) `<dl>` metadata pairs label-over-value. (6) No horizontal overflow at any width; 0 axe violations; gates green.
- **Findings fixed:**
  - **[P0] Latent horizontal overflow** — `.head-grid` (`auto minmax(0,1fr)`) with a flat 44px `--text-3xl` and no `overflow-wrap`. Verified by substitution: an 11-character single-token name produced **87px of document overflow at 320px**; today's "Sole Focus" hides it by breaking into two short words. Fixed with `clamp(2rem,6vw,2.75rem)`, `overflow-wrap: anywhere`, and a stack breakpoint at 34rem. Re-verified: **0**.
  - **[P0] Footer not pinned** — `body` was `display: block` with only `min-height: 100vh`, leaving a measured **1273px** of bare background under the 404 footer at a 1900px viewport. Now `body { display:flex; flex-direction:column }` + `main { flex: 1 0 auto }`.
  - **[P1] Monotone rhythm** — one value site-wide. Now three, applied per relationship. Measured at 1280: `/apps/sole-focus/` went `70,70,70,70,70` → **`24,115,38,38,115,115`**; `/support/` → `38,115,38`; `/apps/` → `115,38`.
  - **[P1] Hero double-pad** — `.hero{padding-bottom}` stacked with `.band + .band{margin-top}` for a measured **141px** void, double every other gap, by accident. Removed; the hero→featured beat is now `band-loose` deliberately.
  - **[P1] Tail void** — `--band-y-lg` tail + footer `--band-y` stacked 166–182px after the least important line on each page. Tail reduced to `--band-y`.
  - **[P2] Prose dead width** — ~400 × 1464px of continuously unused canvas, the largest dead region on the site. Now a two-column composition ≥75rem with the maker's note in the second column (sticky), stacking `column-reverse` below so the note reads as an intro. Measured dead width **400px → 0**. Threshold set at 75rem because at 68rem the aside computed to 205px (~25ch) — cramped, not composed.
  - **[P2] `<dl>` pairs** — `.meta` computed four equal 273px tracks with `dt`/`dd` alternating horizontally, so a label never sat above its value (the existing `dd{margin-top}` proved the intent). Wrapped each pair in a div.
  - **[P2] Single-item grid stretch** — `/privacy/` `.policy-list` computed `1152px 0px 0px`; one card spanned the full rail with `space-between` throwing its title and date **774px** apart. Now `auto-fill` + a 26rem max track + `justify-content: start`.
  - **[P2] Third left edge** — `.transcript` was centred at x=291 on a page where everything else starts at x=59. Left-aligned onto the rail.
  - **[P2] `.about-title { max-width: 16ch }`** left 690px (60%) of the rail dead at 44px type → 22ch.
  - **[P3]** `.is-lead .maker-heading` tied with four other headings at `--text-xl` → `--text-2xl` so the home page's third band has a distinct tier. Four pill paddings (`0.15/0.6`, `0.2/0.6`, `0.3/0.85`, spec `0.25/0.6`) unified. `.download-btn` `min-height: 2.4rem` → `2.5rem` (it contradicted the documented shared 3rem/48px rule). Two `34rem` literals → `--measure`. Corrected a false source comment claiming `.gallery` "breaks out past the prose column" — measured, it matches the rail exactly.
  - **Regression caught during verification:** the new `.prose-aside` as an `<aside>` nested a complementary landmark inside MakerNote's own `<aside>` → `landmark-complementary-is-top-level`. Changed to a `div`; re-verified 0.
- **Deliberate exceptions (not fixed, listed for the owner):**
  - **Card-grid consolidation** — 23 near-identical `heading + body` cells across four grids (`about .values`, `support .cards`, `[slug] .feature-grid`, `privacy .sections`) with four different `minmax()` floors. A real finding, but consolidating into one shared utility is a refactor with visual blast radius across four pages; deferred rather than rushed.
  - **`/` and `/apps/` render near-identical skeletons** at one product. Genuine, but the fix is a design decision about what the catalog page should be, not a spacing change. Re-evaluate at product #2.
  - Left-rail asymmetry with capped measures, the spotlight's `0.85fr/1.15fr` split, the hero's 16/24/32/16 internal cadence, and the mobile "Home"-drop are all deliberate and were left alone.
