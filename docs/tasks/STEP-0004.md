# Task Card — STEP-0004

- **ID:** STEP-0004
- **Type:** feature
- **One outcome:** The site is continuously deployed to a live **GitHub Pages project site** (`https://metekaplangit.github.io/solo-developer-portfolio-website/`) via GitHub Actions, with all internal links base-path-correct. Custom domain is deferred to a later step.
- **Why now:** The user wants a live URL to watch the UI evolve; the custom domain is deferred to end-of-project. A project-site deploy needs a base path, so links must be routed through a `withBase()` helper — establishing the pattern all later UI work uses.
- **Acceptance criteria:** (1) `astro.config.mjs` sets `base` to the repo path and `site` to the github.io origin (documented to revert to `/` + real domain later). (2) A `withBase()` helper prefixes every internal link; no internal link 404s under the base path. (3) `.github/workflows/deploy.yml` builds and deploys to Pages on push to `main` (pinned actions, least-privilege permissions). (4) GitHub Pages is enabled (Actions source); the live URL returns HTTP 200 and navigation works. (5) `npm run build`, `npm run check`, `npm test`, `npm audit`, and `python3 scripts/validate-governance.py` all pass.
- **Non-goals:** Custom domain / Cloudflare DNS / HTTPS-on-domain (deferred to a late step); UI visual redesign (STEP-0005); analytics.
- **Evidence method:** command/log artifact (build with base; deploy workflow run; curl of the live Pages URL returning 200 and base-correct links) + `withBase` unit test.
- **Proof classification:** machine-verifiable (live Pages URL smoke is machine-checkable).
