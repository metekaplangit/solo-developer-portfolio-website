# Task Card — STEP-0002

- **ID:** STEP-0002
- **Type:** feature
- **One outcome:** A reusable product-detail page pattern at `/apps/<slug>/`, generated for every product from the existing content model, and linked from the home and catalog cards.
- **Why now:** The product idea centers on a "reusable app/game detail page pattern," and it is the single biggest gap after M1 — catalog cards currently link nowhere. Each product needs a polished, store-reviewer-ready page with its own privacy/support links.
- **Acceptance criteria:** (1) A detail route is generated for every product (`getStaticPaths`), rendering name, type/status, platforms, summary, longDescription body, features, visible store links, screenshot placeholders (with alt text), and canonical privacy + support links. (2) Home and catalog product cards link to the detail page. (3) Each detail page emits per-product SEO/OpenGraph title + description. (4) A "related products" section shows other products of the same type (self excluded). (5) `npm run build`, `npm run check`, `npm test`, and `python3 scripts/validate-governance.py` all pass; no backend or third-party service added.
- **Non-goals:** Real image-optimization/asset pipeline (screenshots stay honest placeholders — later packet); site-wide visual redesign beyond the detail page; deployment (M4); real product copy.
- **Evidence method:** command/log artifact + automated test (relatedProducts unit test; build route list showing per-product detail pages; runtime screenshot of one detail page).
- **Proof classification:** machine-verifiable.
- **Required checks:** `npm run build`; `npm run check`; `npm test`; `python3 scripts/validate-governance.py`; runtime route smoke.
