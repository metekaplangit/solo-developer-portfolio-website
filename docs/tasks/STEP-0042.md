# Task Card — STEP-0042

- **ID:** STEP-0042
- **Type:** feature (UI — component consistency pass)
- **One outcome:** Every heading+body block on the site is one component with one padding, one radius, one type scale and equal heights per row, and the privacy policies read as documents (one full-width row per section) rather than as a ragged dashboard.
- **Why now:** Owner-annotated screenshots of the live site, three surfaces: **home** — "Weird gap" (hero→featured) and "This looks odd text" (maker note); **privacy overview** — "Text color not matching rest of the pages concept" and "Looks odd all boxes have different size"; **Sole Focus privacy** — "Weird box sizes / All over the place design", plus the explicit direction: *"Create one box a line, make it long width, use bullet points. Apply similar style to other related pages."* Closing note: *"bring more consistency to our whole webpage."* This also lands STEP-0041's deferred exception (card-grid consolidation).
- **Acceptance criteria:** (1) One shared `.card` / `.card-grid` / `.row-stack` system in `global.css`; no page or component defines its own card padding, radius or title size. (2) Cards stretch — zero ragged rows on any page at 1440. (3) Policy sections are one full-width row per line with a left heading rail and bullet lists. (4) `.policy-lead` matches every other page's lede colour. (5) Home hero→featured gap reads as connected; the maker note fills its container. (6) No horizontal overflow at any width; 0 axe violations; gates green.
- **What was wrong (measured):** four card definitions for what is visually one component — paddings `clamp(5,2.5vw,8)` / `clamp(5,2.5vw,6)` / `clamp(4,2vw,6)` / `clamp(5,2.5vw,6)`, radii `--radius-lg` and `--radius`, title sizes `--text-lg` and `--text-base`, and four grid floors (17 / 20 / 22 / 26rem). `PolicyArticle`'s `.sections` also set `align-items: start`, which disabled stretching and produced the visibly ragged heights the owner circled (measured 127/199/127 in one row). `.policy-lead` was `--text` at 20px where every other lede is `--muted`. The home hero carried a `band-loose` 115px separation from a panel that continues its pitch, and the home maker note was a bare 752px text column in a 1152px band.
- **Fixes:** shared `.card` system (one padding, one radius, `.card-title`, `.card-body`, stretch by default, explicit "never set `align-items: start`" note); `.row-stack` full-width variant with a `minmax(0,15rem) minmax(0,1fr)` heading rail at ≥56rem; PolicyArticle sections rebuilt onto it; About values, Support cards and the product feature grid converted (track floor is now the only local override); `.policy-lead` → `--muted`; hero→featured back to the default separation; the home maker note wrapped in a `.panel` and given the same heading-rail composition, so the whole site shares one "label left, content right" pattern.
- **Non-goals:** Copy, palette, imagery, the headline and lead image (owner-directed, in `.impeccable/critique/ignore.md`).
- **Evidence method:** built-markup verification of class adoption; scripted per-row equal-height check at 1440; overflow sweep 320–1440 × 8 routes; axe per route; gates.
- **Proof classification:** manual-runtime (measured) + machine-verifiable.
- **Required checks:** `npm run build`; `npm run check`; `npm test`; validator; CI a11y gate; live verification.
- **Issue links:** Owner screenshot annotations, 2026-07-18. Lands the card-grid consolidation deferred as a deliberate exception in `docs/tasks/STEP-0041.md`.
