# Task Card — STEP-0029

- **ID:** STEP-0029
- **Type:** feature (UI — small-screen usability; external design-review remediation)
- **One outcome:** Comfortable small-screen ergonomics for the site's compact controls: a deliberate narrow-screen header that stops eating a third of the viewport, and touch targets that meet comfortable hit-area sizes — adopting review-0001 findings OPP-01 and OPP-08.
- **Why now:** External design review (`project-designer` review-0001, 2026-07-17) measured the 390px header at ~146px tall (brand row + five wrapped nav links, no narrow-screen mode) and found compact controls with thin touch tolerance (nav links ~33px high; carousel dots 11px with no larger hit area). Verified against source: `Nav.astro` wraps freely; `.dot` is literally 11×11px.
- **Acceptance criteria:** (1) At ≤ ~640px the header keeps the brand on one row and presents the five destinations as a single horizontally scrollable, non-wrapping row with a visible edge-fade affordance — **zero JavaScript** (per ARCHITECTURE guardrails; the review's disclosure-menu option is intentionally not taken). (2) Header height at 390px drops materially (~146px → ≤ ~110px) with no horizontal page overflow. (3) On coarse pointers, nav links and carousel dots present ≥ 40–44px hit areas without changing their visual size (dots stay 11px visually via a centered pseudo-element inside a larger transparent button). (4) Focus rings cover the full interactive area; `aria-current` state remains visible in the scrollable row. (5) A11y gate stays ≥ 0.95; all gates green; verified at 390px and 320px in the preview with screenshots.
- **Non-goals:** A JS hamburger/disclosure menu; redesigning header aesthetics; touching desktop layout; OPP-02 (declined — CTA repetition is the user's explicit intent); OPP-05/06/03 (deferred — taste-sensitive, preview-first).
- **Evidence method:** measured header geometry before/after at 390px (DOM getBoundingClientRect) + screenshots (390px, 320px) + computed hit-area sizes + gates.
- **Proof classification:** manual-runtime (measured geometry + visual) + machine-verifiable (build/tests/validator/a11y gate).
- **Required checks:** `npm run build`; `npm run check`; `npm test`; validator; CI a11y gate; preview measurements.
- **Issue links:** Adopts review-0001 OPP-01 + OPP-08 (report: `project-designer` design-library, review-0001). Declines OPP-02 (user-intent conflict), defers OPP-05/06/03.
- **Rollback/containment:** CSS-only in `Nav.astro` + `ScreenshotShowcase.astro` (+ tiny markup for dot pseudo-element); `git revert -m 1` restores.
- **Evidence after completion:** Released as v0.29.0. 390px header measured **146px → 97px** (no-wrap scrollable nav row with edge fade); dots visually 11px inside **44×44px** buttons; nav links ≥44px tall on coarse pointers; no horizontal overflow at 390/320; focus rings on full hit areas; gates green; deploy through a11y gate; live-verified.
