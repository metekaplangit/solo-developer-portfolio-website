# Task Card — STEP-0036

- **ID:** STEP-0036
- **Type:** feature (UI — wider, image-forward home + reduced text density)
- **One outcome:** The site — especially the home featured area — reads wide, spacious, and image-led instead of a narrow center column of stacked text: a wider shell, a more prominent featured screenshot, and trimmed copy so nothing feels cramped.
- **Why now:** User direction: the featured home area and images look "too cramped up and narrow", "text stacking up on a vertical line", wants things wider, images more prominent, and less text. Confirmed at 1440px: content sits in a 68rem column with large dead margins; the featured spotlight's left column stacks ~8 text blocks (tags/name/3-line lede/3 bullet highlights/CTA/view+privacy/facts/trust). Rival benchmark (Panic, Rogue Amoeba, Things, Raycast) is uniformly wide + big-imagery + minimal-text.
- **Acceptance criteria:** (1) Wider shell: `--maxw` 68rem → ~78rem so the whole site uses more width; hero widened. (2) Featured spotlight: the screenshot is the dominant element (image column meaningfully larger than text), the section is wider and more padded, and the vertical text stack is cut — the 3 bullet highlights are removed and facts/trust compress. (3) Reading columns (`--maxw-prose`) widened modestly for less-cramped prose. (4) No horizontal overflow at 1440/768/390; a11y ≥ 0.95; images keep their aspect ratio and lazy/eager behavior. (5) Real product screenshot only — no fabricated UI. (6) Gates green; verified with before/after screenshots at desktop + mobile; shipped.
- **Non-goals:** Rewriting product description prose (T4 not selected); new content/claims; changing the product content model; per-page bespoke redesigns beyond home + shared width tokens this packet.
- **Evidence method:** before/after preview screenshots at 1440 + 390 (+768 check) + overflow assertions + gates + post-deploy live curl/screenshot.
- **Proof classification:** manual-runtime (measured/visual) + machine-verifiable (build/tests/validator/a11y gate).
- **Required checks:** `npm run build`; `npm run check`; `npm test`; validator; CI a11y gate; live verification.
- **Issue links:** User-directed redesign; aligns with review-0002 cross-lane theme "clearer product understanding, real screenshots are a strength" and USER-08 (product visible sooner). Honors the standing rules: real imagery only; show rendered result.
- **Rollback/containment:** CSS + home markup (+ width tokens); `git revert -m 1` restores. Token change is the widest-reach edit — verified across pages.
