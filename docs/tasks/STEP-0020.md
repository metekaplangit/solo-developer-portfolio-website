# Task Card — STEP-0020

- **ID:** STEP-0020
- **Type:** feature (UX affordance)
- **One outcome:** Make a product's **icon a link to that product** everywhere it appears (not just the title), so clicking the icon opens the item.
- **Why now:** User: clicking an app/game/project icon should open the related item, like the title does.
- **Acceptance criteria:** (1) `ProductAvatar` gains an optional `href` — when set, the avatar becomes a link with `cursor:pointer`, a subtle hover-lift + press, a rounded focus ring, and an `Open <name>` aria-label (image marked decorative inside the link). (2) Icon links to the app detail page on the **home spotlight**, **catalog/home cards**, and the **per-product privacy page**. (3) On the app **detail page** the icon is **not** linked (self page). (4) The clickable icon does not pick up the content-link underline (excluded in `global.css`). (5) The privacy icon's label uses the **product name** ("Open Sole Focus"), not the policy title. (6) `npm run build`, `npm run check`, `npm test`, validator pass; no overflow desktop/mobile; a11y preserved.
- **Non-goals:** Making the whole card clickable (nested-link a11y); linking the detail-page icon to itself.
- **Evidence method:** command/log (build/check/test/validator) + built-HTML assertion that icon-link elements exist on cards/spotlight/privacy and are absent on the detail page + runtime computed-style/aria check + overflow scan.
- **Proof classification:** machine-verifiable + manual-runtime.
- **Required checks:** `npm run build`; `npm run check`; `npm test`; validator; runtime.
- **Issue links:** none.
- **Rollback/containment:** work on `feature/STEP-0020-clickable-icons`; revert the merge or branch from `v0.19.x`; component/route/CSS only.
