# Task Card — STEP-0028

- **ID:** STEP-0028
- **Type:** feature (UI — prominent top-of-page download CTAs)
- **One outcome:** A visitor can find and click a polished Mac App Store download button at the TOP of every relevant page — product detail header, home spotlight, and catalog card — instead of only in the bottom "Get it" section.
- **Why now:** User request post-release: the single download button sits at the bottom of the detail page; people scanning any page should hit a clear download CTA immediately.
- **Acceptance criteria:** (1) A reusable `DownloadButton.astro` renders a branded, accessible download button (Apple glyph + label, accent primary style, `rel="noopener" target="_blank"`) from a product's first available store link — data-driven, nothing hardcoded per page. (2) Placed top-of-page: detail page header block (above the gallery), home spotlight (as the leading CTA), and the catalog card. (3) The bottom "Get it" section remains (no removal). (4) Renders nothing for products without an available link (future products degrade gracefully). (5) A11y: accessible name includes the product ("Download X on the Mac App Store"); Lighthouse a11y gate stays ≥0.95. (6) All gates green; deployed; live pages verified serving the new buttons.
- **Non-goals:** Apple's official badge artwork (kept to the site's own design language); redesigning the description; touching gallery/layout otherwise.
- **Evidence method:** rendered preview screenshots (all three surfaces) + built-HTML grep (button per page, exact URL) + gates + post-deploy live curl.
- **Proof classification:** machine-verifiable (build/tests/validator/a11y gate) + manual-runtime (screenshots/live).
- **Required checks:** `npm run build`; `npm run check`; `npm test`; validator; CI a11y gate; live verification.
- **Issue links:** Follow-on to STEP-0027 (release pass); addresses the discoverability gap it left.
- **Rollback/containment:** one new component + three small insert points; `git revert -m 1` the merge restores prior CTAs.
- **Evidence after completion:** Released as **v0.28.0**. DownloadButton on all three surfaces (detail header, home spotlight leading CTA, catalog card), verified in preview screenshots and on the live site (exact store URL present on /, /apps/, /apps/sole-focus/). Bottom "Get it" intact. Gates: build 8 / check 0-0-0 / 38 tests / validator 43/43; deploy green through the a11y gate. **MC-0014 + ENH-0004** (due at feature step 28) run alongside.
