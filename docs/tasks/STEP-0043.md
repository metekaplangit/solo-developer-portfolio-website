# Task Card — STEP-0043

- **ID:** STEP-0043
- **Type:** feature (UI — home maker note, quiet tone)
- **One outcome:** The home page's "Why this exists" band becomes a row of small, identical, deliberately understated cells that recede behind the featured product instead of competing with it.
- **Why now:** Owner, on the live home page: *"I just do not like how it is presented… Use multiple small boxes here which are perfectly aligned and the same shape and size. But I want them to be very, very subtle so the user's focus must be on the featured application."*
- **Interpretation note:** the request opened with "less subtle, less prominent" and closed with "very, very subtle so the user's focus must be on the featured application." Built to the second, which is stated with emphasis and carries the reason. If the first was meant literally, this is a one-token revert.
- **Acceptance criteria:** (1) Multiple small boxes replacing the single panel. (2) Identical width AND height at every viewport — no orphan stretched cell at any width. (3) Visually subordinate to the featured spotlight. (4) Body text still clears WCAG AA. (5) No horizontal overflow; 0 axe violations; gates green.
- **How "subtle" was achieved:** flatness, not just smaller size. The spotlight is an elevated gradient panel with a shadow and an accent radial wash; cells that merely shrank would still have competed. The cells therefore use the flattest container in the system — transparent fill, a single 1px `--border` hairline, no shadow, no accent — and the heading drops from `--text-2xl` (32px) to a 15px `--faint` label. Single-cue is preserved (outline **or** fill, never both). Recorded in the component so a future edit doesn't reintroduce a surface fill.
- **Non-goals:** The maker-note copy (unchanged, still the approved `makerNote` from the content file); the product page's `inline` tone; the featured panel.
- **Evidence method:** per-width cell dimension equality check (10 widths), overflow sweep, axe, contrast computation, gates.
- **Proof classification:** manual-runtime (measured) + machine-verifiable.
- **Required checks:** `npm run build`; `npm run check`; `npm test`; validator; CI a11y gate; live verification.
- **Issue links:** Owner direction on the v0.39.0 home page; follows STEP-0042.
- **Rollback/containment:** One `tone` prop value plus its scoped styles. `git revert -m 1` restores v0.39.0.
- **Evidence after completion:** Released as **v0.39.1**. Three cells, **identical width and height at all ten tested widths** (320/390/430/600/744/820/960/1024/1280/1440). Two layout bugs were caught and fixed during verification: `auto-fit` left the third cell alone on a second row at 744–820px where `1fr` stretched it to double width (now `grid-auto-flow: column` above 60rem, single column below — no orphan possible at any count); and stacked cells sized to their own copy, so `grid-auto-rows: 1fr` was added to keep them identical when stacked. Body text `--faint` on `--bg` = **5.59:1**, clears AA. 0 overflow across 40 checks; 0 axe violations. Gates: build 8 / check 0-0-0 / 45 tests / validator pass. No checkpoint due (feature #42).
