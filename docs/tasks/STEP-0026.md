# Task Card — STEP-0026

- **ID:** STEP-0026
- **Type:** feature (UI — product screenshot gallery)
- **One outcome:** Replace the vertically-stacked product screenshot showcase with a single-image **gallery/carousel** so a product page stays compact no matter how many screenshots it has (the user expects ~20), keeping the existing large image size.
- **Why now:** User asked directly: the stacked screenshots forced a long scroll to reach the description/highlights; wanted one big image at a time with left/right arrows and a smooth slide, "because I might have 20 screenshots."
- **Acceptance criteria:** (1) One large screenshot visible at a time at the **unchanged** frame size (`max-width: 64rem`, `aspect-ratio: 2880/1800`). (2) Left/right arrow buttons switch shots with a smooth slide; arrows disable at the first/last shot. (3) Works for N shots without added page height (compact regardless of count). (4) Clickable dots + live "n / N" counter; native swipe/trackpad and keyboard (←/→) navigation. (5) Honors `prefers-reduced-motion`; degrades to native swipe with zero JS. (6) Arrows/dots sized for visibility and styled with the site's accent tokens (per follow-up user request). (7) **Accessibility:** page stays **axe-clean** and the Lighthouse a11y gate (≥0.95) passes. (8) `npm run build`, `npm run check`, `npm test`, validator pass; shipped to production.
- **Non-goals:** Changing the screenshot image size/framing; a crossfade instead of slide; a JS-transform slider (kept CSS scroll-snap); lightbox/zoom; touching other components.
- **Evidence method:** command/log (build/check/test/validator) + runtime preview driving the gallery (arrow/dot/keyboard nav, counter, end-state disabling) + **axe-core in-browser audit = 0 violations** + live production verification (metkapstudio.com serves the gallery) + green Lighthouse a11y gate in CI.
- **Proof classification:** manual-runtime (driven gallery + measured axe audit) + machine-verifiable (build/tests/validator/CI a11y gate).
- **Required checks:** `npm run build`; `npm run check`; `npm test`; validator; runtime preview; CI Lighthouse a11y gate.
- **Issue links:** resolves the user-reported "too much scrolling past stacked screenshots" request; follow-up requests (bigger/more-visible arrows + bigger dots, on system colors) applied.
- **Rollback/containment:** component-scoped (`src/components/ScreenshotShowcase.astro`); `git revert -m 1` the release merge, or revert the individual gallery/a11y merges; other pages unaffected.
- **Evidence after completion:** Released as **v0.26.0**. Gallery shipped to production at https://metkapstudio.com/apps/sole-focus/ (HTTP 200, gallery markup live). Driven in preview: arrows + ←/→ keyboard + dots page one slide at a time, counter updates, prev/next disable at ends, 5 shots. Initial deploys correctly **blocked** at Lighthouse a11y 0.91; fixed three carousel axe violations (`aria-hidden-focus`, `aria-required-children`, `scrollable-region-focusable`) — re-audited **axe-clean (0 violations)** and the a11y gate passed (CI run 28855465208). `npm test` → 36 passed; `npm run check` → 0/0/0; `npm run build` → 8 routes; validator → 40/40.
